Simulador Banxico :: Sistema Multi-Agente

Simulación de la Junta de Gobierno del Banco de México

Aplicación web multi-usuario que reproduce la deliberación de política monetaria del Banxico — cinco agentes con posturas diferenciadas debaten en tiempo real, votan la decisión en puntos base y publican la minuta oficial en Markdown.

5 AGENTES
Gobernadora + 4 Subgobernadores
3 MODOS
Chat · Junta · Mapa Mundial
4 TOOLS
snapshot · search · calc · history
5 APIS
Banxico · FRED · Yahoo · WB · Tavily

01 — Flujo de una junta

Pipeline en 5 fases

Cada junta sigue un flujo estructurado, transmitido por WebSocket al frontend en tiempo real. El usuario plantea el tema, los agentes argumentan con datos en vivo, votan en puntos base y un Secretario redacta la minuta en Markdown.

FASE 01
Apertura
La Gobernadora plantea el contexto y el tema sometido a deliberación
FASE 02
Posturas
Cada Subgobernador presenta su lectura usando datos macro en vivo
FASE 03
Debate
Rondas de réplica con tool-calling visible y caja compartida de resultados
FASE 04
Votación
Voto formal en bps con desempate por la Gobernadora
FASE 05
Minuta
Secretario redacta la minuta en Markdown + memoria persistente por agente

02 — Composición

Los 5 miembros de la Junta

Cada persona tiene un system_prompt propio, postura base y memoria persistente. Lo aprendido en chats se traslada a las juntas y viceversa: facts durables + meeting_summary en primera persona, almacenados en SQLite a nivel agent_id.

03 — Tecnología

Stack tecnológico

Aplicación web full-stack en una sola imagen Docker. Backend FastAPI con orquestación LangChain; frontend React + Vite empaquetado como estáticos y servido por el mismo proceso. SQLite local persistido en un volumen. Caddy enfrente para TLS automático en producción.

Backend
FastAPI + LangChain 0.3
Lifespan + WebSocket nativo. Cada turno del debate es un agente LangChain con herramientas tipadas. Switch en runtime entre Anthropic SDK y OpenRouter sin cambiar el código del runtime.
LLM Backend
Claude / OpenRouter
PROVIDER=anthropic|openrouter. Modelos por defecto: Claude Sonnet 4.6 vía Anthropic, o cualquier slug de OpenRouter (Sonnet 4.6, Kimi K2.6, etc.). Function calling para las 4 herramientas.
Persistencia
SQLite + SQLAlchemy 2.0
Schema declarativo (users, agents, meetings, messages, votes, agent_memory). Memoria compartida entre chat y juntas a nivel agent_id. Volumen Docker en /app/data.
Frontend
React 18 + Vite 5
TypeScript 5.6, Tailwind 3.4, TanStack Query, react-router-dom. Streaming en vivo del debate por WebSocket. Mapa mundial con ECharts cargado desde CDN.
Auth
JWT + bcrypt
Login por usuario/contraseña con bcrypt; token JWT en header Authorization para HTTP y ?token=... para WebSocket. Registro abierto controlable con ALLOW_REGISTRATION.
Despliegue
Docker + Caddy
Multi-stage Dockerfile (Node build → Python serve). docker-compose.prod.yml + Caddyfile con ACME automático. deploy.sh inyecta commit SHA y build time consumibles en GET /api/version.

Capas de la arquitectura

[PRESENTACIÓN] React SPA | Tailwind | ECharts (mapa) | [API] FastAPI ── WebSocket ── REST ── JWT auth | [ORQUESTACIÓN] LangChain runtime → apertura → debate → votación → minuta | [AGENTES] Méndez :: Aguirre :: Robles :: Carrillo :: Vega | [TOOLS] get_macro_snapshot · web_search · calculator · consult_banxico_history | [DATOS] Banxico SIE · FRED · Yahoo Finance · World Bank · Tavily | [ESTADO] SQLite (users, agents, meetings, messages, votes, agent_memory)

Herramientas LangChain expuestas al agente

[snapshot]
get_macro_snapshot
Snapshot consolidado: tasa Banxico, Fed Funds, INPC headline/subyacente, USD/MXN, PIB, desempleo, WTI, Brent, Mezcla MX, CPI USA/EZ. Caché 5 min.
[search]
web_search
Búsqueda web vía Tavily. Devuelve hasta 5 resultados (título, URL, extracto). Si falta la key, mensaje claro y el agente sigue con el resto.
[calc]
calculator
Aritmética segura con numexpr: diferenciales de tasa, porcentajes y conversiones simples. Útil para no inventar cifras derivadas.
[history]
consult_banxico_history
Búsqueda por keywords sobre minutas oficiales (mar-2026, may-2026) almacenadas en backend/app/data/banxico_history/. Sin red trip.

05 — Uso

Tres modos de operación

Multi-usuario: cada quien inicia sesión y queda como creador de sus chats y juntas — la lista es compartida y se ve quién creó cada uno. El registro abierto se cierra con ALLOW_REGISTRATION=false.

01
Chat 1-a-1
Conversación uno-a-uno con cualquier agente. Cada 6 mensajes del usuario el extractor escribe hasta 3 facts durables (preferencias, compromisos, datos clave) en la memoria del agente.
facts persistentes WebSocket
02
Simulación de Junta
Debate de los 5 agentes con número configurable de rondas. Al cierre, un meeting_summary en primera persona se escribe en la memoria de cada participante.
multi-agente minuta MD
03
Mapa Mundial
Visualización ECharts coloreada por inflación, PIB, deuda externa, deuda pública o tensión política. Incluye cuellos de botella del petróleo y lista curada de conflictos 2026.
World Bank EIA · conflictos

Capacidades transversales

[mem]
Memoria compartida
Lo aprendido en chat se traslada a la junta y viceversa. Cada turno carga últimos 10 facts + 3 meeting_summaries como SystemMessage.
[live]
Streaming WebSocket
Tokens, tool-calls y resultados llegan al frontend en vivo. Visible en ToolCallTrace para auditar qué consultó cada agente.
[ver]
Versión auditable
GET /api/version expone commit SHA, fecha del commit y build time. El footer del sitio muestra "desplegada hace X" con tooltip.
[tls]
TLS automático
Caddy + Let's Encrypt en producción. ACME challenge HTTP-01 al primer arranque; certs reusados desde el volumen caddy_data.
